Implement Transaction Security Layer for UStack.

OVERVIEW

All movement of funds inside UStack must require user authorization.

No funds may be moved without explicit user approval.

This applies to:

* Withdrawals
* Deposits into vaults
* Transfers from vault to available balance
* Vault unlocking
* Mobile Money withdrawals
* Lightning withdrawals
* Move to Hardware Wallet
* Any future fund movement feature

AUTHORIZATION METHODS

Support two authorization methods:

1. Transaction PIN
2. Biometric Authentication

Users may enable one or both.

If biometric authentication is available and enabled:

Prompt fingerprint .

If biometric authentication is unavailable:

Require Transaction PIN.

TRANSACTION PIN

Create secure Transaction PIN functionality.

Requirements:

* 4 digits
* Separate from login system
* Stored using strong hashing
* Never stored in plain text
* Never returned by APIs

Database:

Add fields to profiles table:

transaction_pin_hash
transaction_pin_enabled
transaction_pin_created_at
transaction_pin_updated_at

SECURITY REQUIREMENTS

Never store raw PINs.

Use bcrypt or Argon2 hashing.

Never expose authorization data to frontend.

Validate authorization server-side.

Prevent brute-force attempts.

Prevent replay attacks.

Use secure session validation.
